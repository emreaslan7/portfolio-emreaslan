# Add Blog Post — Project Workflow

## 1. Image

Place the image in `public/articles/<filename>.png`.  
Reference it in frontmatter as `image: '/articles/<filename>.png'`.

## 2. MDX file → `content/<slug>.mdx`

Required frontmatter (order preserved):

```yaml
---
title: 'Post Title'
publishedAt: 'YYYY-MM-DD'
author: 'Emre Aslan'
summary: 'One or two sentences describing the post.'
image: '/articles/<filename>.png'
---
```

- `slug` must be kebab-case (e.g. `what-are-soulbound-tokens`).
- `publishedAt` is a date string only (no time).
- `author` defaults to `'Emre Aslan'`.
- `image` is optional but should match the PNG filename.
- Do NOT include `content` or `mdx` fields — those are generated by content-collections.

## 3. Content conventions

- Use `#` for the main heading (not `##`).
- Use `<div align="center">` + `![](url)` for centered images.
- Code blocks use standard markdown triple-backticks with language tag (e.g. ` ```solidity `).
- Code blocks can include `title="..."` metadata in the opening fence for a title bar.

## 4. No other files to touch

- `content-collections.ts` — no changes needed (wildcard covers all `content/*.mdx`).
- `src/data/resume.tsx` — no changes needed.
- Generating the build (`pnpm build`) will regenerate `.content-collections/generated/` automatically.

## 5. Image naming convention

All existing blog images follow the pattern `articleXX.png` where XX is sequential (02, 06, etc.). Follow existing sequence or use a descriptive slug name — whichever matches project precedent.
